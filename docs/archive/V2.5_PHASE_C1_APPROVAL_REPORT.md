# V2.5 Phase C1 — Human Approval Layer 实现报告

## 概述

为 Task Execution 增加人工审批机制。当 Executor 遇到标记为需要审批的步骤时，自动暂停 Task 并创建 Approval Request，等待人工审批后继续或终止。

## 约束

- ❌ 不开发 Planner
- ❌ 不修改 MAEOS
- ❌ 不改变 Chat 流程

## 数据模型

### TaskApprovalModel

```python
class TaskApprovalModel(Base):
    __tablename__ = "task_approvals"

    id           : str     # UUID PK
    task_id      : str     # FK → tasks.id (CASCADE)
    step_id      : str?    # FK → task_steps.id (SET NULL)
    status       : str     # PENDING | APPROVED | REJECTED | EXPIRED
    reason       : str     # 审批原因/备注
    requested_at : datetime
    approved_at  : datetime?
    approved_by  : str?    # 审批人标识
```

### ApprovalStatus 枚举

```
PENDING  →  APPROVED  (批准 → resume task)
PENDING  →  REJECTED  (拒绝 → cancel task)
PENDING  →  EXPIRED   (过期 → 不修改 task 状态)
```

### TaskStepModel 新增字段

```
requires_approval: str  # "0" = 不需要审批, "1" = 需要审批
```

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/tasks/{task_id}/approvals` | 列出任务的审批请求 (可选 `?status=`) |
| `GET` | `/api/tasks/approvals/{approval_id}` | 获取单个审批请求 |
| `POST` | `/api/tasks/approvals/{approval_id}/approve` | 批准 → resume task |
| `POST` | `/api/tasks/approvals/{approval_id}/reject` | 拒绝 → cancel task |

请求体（approve/reject）：
```json
{
  "approved_by": "admin",
  "reason": "Looks good"
}
```

## 执行流程

```
Executor._execute_single_step()
  │
  ├─ 检查 step.requires_approval == "1" ?
  │    ├─ No  → 正常执行 (MAEOS submit)
  │    └─ Yes → TaskApprovalService.create_approval()
  │               ├─ 暂停 Task (EXECUTING → PAUSED)
  │               ├─ 创建 PENDING Approval
  │               ├─ 发射 APPROVAL_REQUIRED 事件
  │               └─ raise ApprovalRequiredError (Executor 捕获，非失败)
  │
  └─ 用户调用 approve/reject API
       ├─ 批准 → PAUSED → EXECUTING, Approval → APPROVED
       └─ 拒绝 → PAUSED → CANCELLED, Approval → REJECTED
```
