## 2. Retry Support

### Integration with `runtime/retry_policy.py`

RetryPolicy already existed — now wired into TaskExecutor.

**Flow:**
```
Step FAILED → RetryPolicy.classify(error) → FailureType
  → VALIDATION_FAIL → RETRY
  → LOGIC_FAIL → FALLBACK (abort)
  → SYSTEM_FAIL → ABORT (no retry)
  → UNKNOWN → RETRY

On RETRY:
  1. Update step FAILED
  2. Increment retry_count
  3. Backoff delay (configurable: fixed/linear/exponential)
  4. Reset step to PENDING
  5. Retry with attempt += 1

On ABORT:
  1. Mark step FAILED (permanently)
  2. Mark task FAILED
  3. Raise RuntimeError
```

**Default RetryPolicy:**
- max_attempts: 2 (1 initial + 1 retry)
- backoff: linear
- base_delay_ms: 2000

### Step retry_count tracking
- `TaskStepModel.retry_count` incremented on each retry
- Preserved across retry cycles

## 3. Execution Cost Tracking

### New Fields on TaskExecutionModel

| Field | Type | Description |
|-------|------|-------------|
| `input_tokens` | Integer | Estimated input tokens (text length / 4) |
| `output_tokens` | Integer | Estimated output tokens |
| `total_tokens` | Integer | Sum of input + output |
| `estimated_cost` | Integer | Micro-dollars (µ$) — $0.001 = 1000 µ$ |

### Token Estimation

```python
estimate_tokens(text) → len(text) // 4 (min 1)
```

### Cost Estimation

Default rates (configurable):
- Input: $0.001 / 1K tokens
- Output: $0.002 / 1K tokens

```python
cost_µ$ = (input_tokens / 1000 * input_rate) + (output_tokens / 1000 * output_rate)
```

`record_execution_with_cost()` — one-call update with auto-estimation.
