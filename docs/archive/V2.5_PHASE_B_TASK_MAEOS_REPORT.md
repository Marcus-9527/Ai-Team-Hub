## Architecture

```
POST /api/tasks/{id}/execute
         │
         ▼
┌─────────────────────────────────┐
│        routes/tasks.py          │
│  - Transition PLANNING→EXECUTING│
│  - Get MAEOS instance           │
│  - Call TaskExecutor            │
└──────────┬──────────────────────┘
           │
           ▼
┌─────────────────────────────────┐
│    TaskExecutor.execute_task()  │
│  - Iterates PENDING steps       │
│  - For each step:               │
│    1. Build context             │
│    2. Submit to MAEOS           │
│    3. Create Execution record   │
│    4. Wait for MAEOS            │
│    5. Record result             │
│    6. Update step status        │
│  - Final: COMPLETED / FAILED    │
└─────────────────────────────────┘
```

## New Files

### `backend/services/task/task_context.py` — Task Context Builder
- `build_step_context()` — builds input for each step:
  - Task goal + current step objective
  - Outputs from prior completed steps
- `build_maeos_description()` — returns the full context string for MAEOS.submit()

### `backend/services/task/task_result.py` — Task Result Handler
- `record_execution()` — creates TaskExecution DB record
- `update_execution_result()` — updates with output/duration/trace
- `handle_step_success()` — updates step output, transitions to COMPLETED
- `handle_step_failure()` — records error, transitions to FAILED
- `handle_task_completion()` / `handle_task_failure()` — task-level status

### `backend/services/task/task_executor.py` — Task Executor
- `execute_task()` — main entry: validates state, iterates pending steps
- `_execute_single_step()` — submit to MAEOS → wait → record result
- `get_task_progress()` — returns step counts and details

## Route Updates (routes/tasks.py)

### New Endpoints
| Method | Path | Purpose |
|--------|------|---------|
| POST | `/api/tasks/{id}/steps` | Add a step |

| GET | `/api/tasks/{id}/steps/{step_id}` | Get step detail |
| PATCH | `/api/tasks/{id}/steps/{step_id}` | Update step |
| GET | `/api/tasks/{id}/progress` | Execution progress |
| POST | `/api/tasks/{id}/executions` | List executions for a step |

### Modified Endpoints
- `POST /api/tasks/{id}/execute` — now **actually runs** steps through MAEOS
  - Transition PLANNING → EXECUTING
  - Fetches MAEOS singleton
  - Calls `TaskExecutor.execute_task()`
  - Returns `TaskDetailResponse` (with step results)

## Execution Flow

```
POST /api/tasks/ → create task (CREATED)
POST /api/tasks/{id}/steps → add steps (PENDING)
POST /api/tasks/{id}/plan → CREATED → PLANNING
POST /api/tasks/{id}/execute → PLANNING → EXECUTING → run steps

  Step 1: PENDING → RUNNING → submit to MAEOS → wait → COMPLETED
  Step 2: PENDING → RUNNING → submit to MAEOS → wait → COMPLETED

  Task: EXECUTING → COMPLETED (or FAILED on any step error)
```
