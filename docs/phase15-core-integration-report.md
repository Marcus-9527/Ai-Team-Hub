## Data Flow Changes

### Before (Phase 14)

```
Human message → TeamCollaboration → build_turn_prompt → ROLE_AXIS_PROMPTS → LLM
                                                          ↑ (无 BrainLoader)

Task → executor._run_task → agent.py / reviewer.py
                             → teammate.system_prompt (无 BrainLoader)

Human message → TeamCollaboration → generate_team_response
                                     ↑ (无 Cede Protocol)

Human message → TeamCollaboration → LLM → 直接发送
                                           ↑ (无 Policy Gate)
```

### After (Phase 15)

```
Human message → Policy Gate ─→ Cede Protocol ─→ TeamCollaboration
                  │                │                 │
                  ↓  blocked       ↓  cede           ↓ respond
               (excluded)       "chose not to    build_turn_prompt
                                  respond"          │
                                                    ↓
                                              BrainLoader.build_prompt()
                                                    │
                                                    ↓
                                              ROLE_AXIS_PROMPTS
                                                    │
                                                    ↓
                                               LLM

Task → executor._run_task
        → teammate["system_prompt"] ← BrainLoader.build_prompt()
        → agent.py / reviewer.py (自动继承 brain 内容)
```

### 关键收敛点

- **chat Engineer == task Engineer**：两条路径最终都通过 `teammate["system_prompt"]` 包含相同的 BrainLoader 内容
  - chat 路径：`stream_teammate()` → BrainLoader → `build_turn_prompt(brain_prompt=...)`
  - task 路径：`executor._run_task()` → BrainLoader → `teammate["system_prompt"]` → `run_engineer_workflow()` / `run_reviewer_workflow()`
- **Cede Protocol 非静默**：CEDE/IGNORE 队友在 SSE 流中显示 `**Name** chose not to respond (role)`，决策持久化到 memory
- **Policy Gate 可扩展**：`check_message_policy()` 当前始终 allow，但接口已就绪，后续可在该函数内添加角色黑名单、频道白名单、速率限制等规则
