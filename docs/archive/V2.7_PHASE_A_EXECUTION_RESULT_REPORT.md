## 1. 新增内容

### 1.1 ExecutionResultModel (`backend/models.py`)

新增数据模型 `execution_results` 表，包含以下字段分组：

| 分组 | 字段 | 类型 | 说明 |
|------|------|------|------|
| **标识** | `id` | TEXT PK | UUID |
| **关联** | `task_step_id` | TEXT FK→task_steps | CASCADE 删除 |
| | `task_execution_id` | TEXT FK→task_executions | CASCADE 删除 |
| **Outcome** | `outcome` | TEXT | SUCCESS / FAILURE / PARTIAL / SKIPPED（带索引） |
| **质量评分** | `completeness` | FLOAT | 0.0–1.0 |
| | `coherence` | FLOAT | 0.0–1.0 |
| | `accuracy` | FLOAT | 0.0–1.0 |
| | `overall_quality` | FLOAT | 0.0–1.0 |
| **计划偏差** | `plan_matched` | TEXT | NONE / MINOR / MAJOR / CRITICAL |
| | `plan_deviation_detail` | TEXT | 偏差描述 |
| **失败分类** | `failure_category` | TEXT | SYSTEM / LOGIC / QUALITY / POLICY / UNKNOWN |
| | `failure_subcategory` | TEXT | 二级分类 |
| | `is_recoverable` | TEXT | "0" / "1" |
| **评估元数据** | `evaluator` | TEXT | llm / rule / manual |
| | `evaluation_confidence` | FLOAT | 0.0–1.0 |
| **生命周期** | `status` | TEXT | CREATED→EVALUATED→COMPARED→CLOSED/REPLAN_TRIGGERED |
| | `replan_triggered` | TEXT | "0" / "1" |
| | `replan_scope` | TEXT | "" / STEP / TASK / CONTEXT |
| | `created_at` | DATETIME | |
| | `updated_at` | DATETIME | onupdate |

同时新增辅助状态类：
- `ExecutionOutcome` — 结果状态常量
- `ExecutionResultStatus` — 生命周期状态常量
- `PlanMatchSeverity` — 偏差严重度常量
- `FailureCategory` — 失败分类一级类别常量

### 1.2 ExecutionResultService (`backend/services/task/task_execution_result.py`)

| 方法 | 签名 | 说明 |
|------|------|------|
| `create_result()` | `(db, **fields) → ExecutionResultModel` | 创建新结果记录 |
| `get_result()` | `(db, result_id) → Optional[ExecutionResultModel]` | 按 ID 查询 |
| `list_results()` | `(db, *, task_step_id, task_execution_id, outcome, status, limit, offset) → list` | 多条件筛选列表 |
| `update_result()` | `(db, result, **fields) → ExecutionResultModel` | 部分字段更新 |
| `count_results()` | `(db, *, task_step_id, outcome, status) → int` | 计数查询 |

Phase A 范围仅限 CRUD，不包含 Evaluation/Classification/Replan 逻辑。

### 1.3 数据库注册 (`backend/database.py`)

`init_db()` 中已导入 `ExecutionResultModel`，数据库初始化时自动创建 `execution_results` 表。
