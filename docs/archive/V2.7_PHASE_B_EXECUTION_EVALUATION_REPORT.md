### 1.2 ExecutionEvaluator 接口 (`base.py`)

```python
class ExecutionEvaluator(ABC):
    @abstractmethod
    async def evaluate(
        self,
        *,
        actual_output: str,
        expected_output: str = "",
        objective: str = "",
    ) -> EvaluationResult: ...
```

### 1.3 EvaluationResult 数据类 (`base.py`)

| 字段 | 类型 | 说明 |
|------|------|------|
| `completeness` | float | 0.0–1.0，输出覆盖度 |
| `coherence` | float | 0.0–1.0，输出结构质量 |
| `accuracy` | Optional[float] | Phase B 保留 None（留待 LLM Judge） |
| `overall_quality` | float | 加权综合 (completeness×0.6 + coherence×0.4) |
| `confidence` | float | 规则评估固定为 1.0 |
| `evaluator` | str | 策略标识，固定 "rule" |
| `details` | dict | 评分分解详情 |

### 1.4 RuleBasedEvaluator 评估策略 (`rule_evaluator.py`)

**completeness (0.0–1.0):**
| 条件 | 方法 | 说明 |
|------|------|------|
| 有 expected_output | 关键词重叠率 | 提取 reference 关键词（≥3字符，去停用词），计算在 actual_output 中出现比例 |
| 仅有 objective | 关键词重叠 ×1.2 | 放宽匹配（上限 1.0） |
| 无 reference | 长度代理 | `<50 chars→0.2`, `<200→0.5`, `<500→0.65`, `≥500且有段落→0.8`, 其他→0.7 |

**coherence (0.0–1.0):**
| 信号 | 加分 | 阈值 |
|------|------|------|
| 非空基础分 | +0.1 | 总有 |
| ≥2 句子 | +0.15 | |
| ≥5 句子 | +0.1 | |
| 有段落分隔 | +0.15 | `\n\n` |
| ≥3 段落 | +0.1 | |
| 平均句长 8–40 词 | +0.1 | 合理句长 |
| 句末标点规范 ≥80% | +0.1 | `. ! ?` 结尾 |
| 句末标点 ≥50% | +0.05 | |
| 有逻辑连接词 | +0.1 | therefore, however, 等 |
| ≥3 逻辑连接词 | +0.1 | |

**overall_quality = completeness × 0.6 + coherence × 0.4**

**accuracy:** None（预留字段）

## 2. evaluate_result() 流程 (`task_execution_result.py`)

在 `ExecutionResultService` 中新增 `evaluate_result()` 方法：

```
ExecutionResult.status == CREATED (或 EVALUATED)
        │
        ▼
outcome in (SKIPPED, FAILURE)? ──Yes──→ 设置 status=EVALUATED, 分数保持 0.0
        │
       No
        │
        ▼
  RuleBasedEvaluator.evaluate(actual_output, expected_output, objective)
        │
        ▼
  update_result() → 写入 completeness/coherence/overall_quality/evaluator/confidence
        │
        ▼
  status = EVALUATED
        │
        ▼
  返回已更新的 ExecutionResult
```
