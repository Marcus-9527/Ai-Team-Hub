# Artifact System — 架构文档 (v1.0)

## 设计原则

- **产物管理即 metadata + 文件存储。** 不做 S3/MinIO/Object Store——当前规模 SQLite + 本地文件系统足够。
- **无自动钩子。** Service 是纯 API；谁产生产物，谁调 `create_artifact()`。ExecutionRuntime / Chat 通过调用 Service 关联产物。
- **YAGNI 优先。** 无审计日志、无版本管理、无 blob 去重引擎。将来按需加。

## 数据流

```
Producer (Executor / Chat / Task)
    │
    ▼
ArtifactService.create_artifact(content, name, type, task_id, ...)
    │
    ├─ content → data/artifacts/<artifact_id>    (filesystem)
    └─ metadata → artifacts 表                    (SQLite)
    │
    ▼
GET /api/artifacts          → list
GET /api/artifacts/{id}     → metadata JSON
GET /api/artifacts/{id}/content → raw text
```

## 文件布局

```
backend/
  services/artifact/__init__.py   ← ArtifactService (CRUD + file I/O)
  routes/artifacts.py             ← FastAPI router
  models.py                       ← ArtifactModel (SQLAlchemy)
data/
  aiteamhub.db                    ← artifacts table
  artifacts/                      ← artifact content files
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | `art_` + uuid hex[:12] |
| task_id | str? | 关联任务 |
| execution_id | str? | 关联执行 |
| type | str | file / code / image / text — 筛选用 |
| name | str | 显示名（带扩展名） |
| path | str | 磁盘绝对路径 |
| content_hash | str | SHA-256[:16] |
| meta | JSON | 任意附加数据（size, language, mime 等） |
| created_at | datetime | UTC |
