# Evaluation System 架构文档 (v1.0)

## 概览

Phase 6 将 Evaluation System 拆分为四个层级：

```
┌─────────────────────────────────────────────────┐
│                  API Layer                       │
│  GET /api/evaluations/{execution_id}            │
│  GET /api/evaluations/stats                     │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────┐
│            EvaluationService                     │
│  evaluate(exec_id, record) → EvaluationRecord   │
│  get(exec_id) / stats()                         │
│  自动 upsert 语义                                │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────┐
│              ScoringEngine                       │
│  score(record_dict) → {latency, cost, artifact, │
│                        error, overall}           │
│  纯函数，无 I/O                                  │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────┐
│         DB: evaluation_records                   │
│  id|execution_id|score|status|latency_score|    │
│  cost_score|artifact_score|error_score|feedback  │
└─────────────────────────────────────────────────┘
```

## 评分模型

| 维度 | 算法 | 权重 | 说明 |
|------|------|------|------|
| latency_score | duration_ms ≤5s→1.0, ≥60s→0.0, 线性插值 | 0.30 | 执行耗时反转评分 |
| cost_score | cost_micro_usd ≤500→1.0, ≥100000→0.0, 对数插值 | 0.25 | 成本反转评分（对数级差） |
| artifact_score | 是否有 artifact tool_call 事件 | 0.20 | 二进制：有→1.0, 无→0.0 |
| error_score | error 是否非空 | 0.25 | 无错误→1.0, 有→0.0 |

**Overall**: `latency * 0.30 + cost * 0.25 + artifact * 0.20 + error * 0.25`

## DB 模型

表名: `evaluation_records`
- `id` — PK (UUID string)
- `execution_id` — FK → execution_records, UNIQUE
- `score` — Float, overall 0–1
- `status` — "PENDING" | "EVALUATED"
- `latency_score`, `cost_score`, `artifact_score`, `error_score` — Float 0–1
- `feedback` — Text, 自动生成的评价文本
- `created_at` — DateTime

## 运行时集成

`ExecutionRuntime._run_task()` 在执行完成或失败后自动调用 `_auto_evaluate()`：

```
set_completed() → _auto_evaluate(exec_id, execution.to_dict())
set_failed()    → _auto_evaluate(exec_id, execution.to_dict())
```

_auto_evaluate 是 fire-and-forget（asyncio.ensure_future），不阻塞执行流。

## Ponytail 简化标记

1. `artifact_score` 是二进制判断，当 artifact 元数据丰富后可升级为类型加权
2. 全局 `_eval_service` 单例，后续可以注入配置
3. `error_score` 只区分有无错误，不按错误长度/类型分级
