## 数据模型

### TaskPolicyModel

```python
class TaskPolicyModel(Base):
    __tablename__ = "task_policies"

    id                : str     # UUID PK
    task_id           : str     # FK → tasks.id (CASCADE, UNIQUE)
    approval_required : str     # "0"|"1"
    max_retry         : int     # 最大重试次数 (默认 2)
    max_cost          : int     # 最大开销(分), 0=无限制
    risk_level        : str     # LOW|MEDIUM|HIGH
    allowed_teammates : str     # JSON array
    created_at        : datetime
```

### RiskLevel 枚举

| 级别 | 行为 |
|------|------|
| `LOW` | 自动放行 (除非 approval_required="1") |
| `MEDIUM` | 若 approval_required="1" → 需审批 |
| `HIGH` | 直接阻断 (POLICY_BLOCKED) |

## Policy 评估流程

```
Executor._execute_single_step()
  │
  ├─ Policy.evaluate_step(step)
  │    ├─ 获取任务 Policy (不存在则创建默认: LOW)
  │    ├─ HIGH   → allowed=False, blocked_reason="risk_level=HIGH"
  │    ├─ retry_count >= max_retry → blocked
  │    ├─ teammate 不在 allowed_teammates → blocked
  │    ├─ MEDIUM + approval_required=1 → requires_approval=True
  │    └─ LOW    → allowed=True (除非 approval_required=1)
  │
  ├─ 结果 not allowed → POLICY_BLOCKED 事件 + PolicyBlockedError
  ├─ 结果 requires_approval → 走 Phase C1 审批流程
  └─ 结果 allowed + no approval → 正常执行
```

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/tasks/{task_id}/policy` | 获取策略 (无则自动创建默认) |
| `PUT` | `/api/tasks/{task_id}/policy` | 更新策略字段 (部分更新) |

请求体示例：
```json
{
  "risk_level": "HIGH",
  "max_retry": 5,
  "max_cost": 1000,
  "approval_required": "1",
  "allowed_teammates": "[\"alice\"]"
}
```

## 默认策略

创建 Task 时自动生成的默认 Policy：

```json
{
  "approval_required": "0",
  "max_retry": 2,
  "max_cost": 0,
  "risk_level": "LOW",
  "allowed_teammates": []
}
```
