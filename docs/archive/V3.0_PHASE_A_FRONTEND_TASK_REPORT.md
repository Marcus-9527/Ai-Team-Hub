# V3.0 PHASE A — Frontend Task Experience

## 概述

使 React 前端与 v2.5+ Task Runtime 后端匹配，新增任务管理完整视图，不修改 Chat 或后端逻辑。

## 新增文件

| 文件 | 类型 | 说明 |
|------|------|------|
| `services/api/task.js` | API Client | 封装全部 task/step/approval/plan/policy REST 端点 |
| `services/taskEventBus.js` | 全局事件总线 | SSE task 事件的发布/订阅，解耦 ChannelView 和 TaskContext |
| `services/taskContext.jsx` | React Context Provider | 全局任务状态管理，含自动轮询和 SSE 订阅 |
| `components/Task/TaskListView.jsx` | 视图组件 | 任务列表，支持状态筛选、创建、删除 |
| `components/Task/TaskDetailView.jsx` | 视图组件 | 任务详情，含元信息、步骤时间线、进度条、操作按钮 |

## 修改文件

| 文件 | 改动 |
|------|------|
| `App.jsx` | 添加 `showTasks` 状态、`TaskProvider` 包裹、TaskListView 路由、侧边栏 props |
| `components/Sidebar/Sidebar.jsx` | 新增 `ListTodo` 按钮、"任务"导航入口 |
| `components/Channel/ChannelView.jsx` | SSE 流中检测 task 事件并转发到 `taskEventBus`，不渲染为聊天气泡 |

| `i18n/en.js` | 同上，英文版 |

## 架构设计

```
SSE Stream (/api/messages/{channelId})
  │
  ├─ teammate_message  → ChannelView (聊天气泡，不变)
  ├─ teammate_end      → 无操作
  ├─ task_started      ─┐
  ├─ step_started       ├─→ taskEventBus.dispatch() ─→ TaskContext (刷新)
  ├─ step_completed     │
  ├─ task_completed     │
  └─ approval_required  ┘
                          → TaskListView / TaskDetailView (自动更新)

REST API (/api/tasks/*)
  │
  ├─ GET /api/tasks              → TaskContext (轮询)
  ├─ GET /api/tasks/{id}         → TaskDetailView (手动/事件)
  ├─ POST /api/tasks/{id}/plan   → 操作按钮
  ├─ POST /api/tasks/{id}/execute → 操作按钮
  └─ ... (全部 CRUD + lifecycle)
```

## Task API Client (`services/api/task.js`)

覆盖全部后端 task 端点：

- **CRUD**: `createTask`, `listTasks`, `getTask`, `updateTask`, `deleteTask`
- **生命周期**: `planTask`, `executeTask`, `pauseTask`, `resumeTask`, `cancelTask`, `completeTask`, `failTask`
- **步骤**: `addStep`, `listSteps`, `getStep`, `updateStep`
- **进度**: `getTaskProgress`, `listExecutions`
- **审批 (Phase C1)**: `listApprovals`, `approveApproval`, `rejectApproval`
- **计划 (Phase C)**: `getTaskPlan`, `applyTaskPlan`
- **策略 (Phase C2)**: `getTaskPolicy`, `updateTaskPolicy`

## TaskContext (`services/taskContext.jsx`)

- `tasks`, `total`, `loading`, `error` — 全局状态
- `selectedTaskId`, `activeFilter` — 选中和筛选
- `refreshTasks()` — 手动刷新
- `setFilter(filter)` — 按状态筛选 (CREATED/EXECUTING/COMPLETED/FAILED)
- `selectTask(id)` / `clearSelection()` — 导航到详情
- 每 10 秒自动轮询
- 订阅 `taskEventBus` 实时更新

## SSE Task 事件流 (`taskEventBus.js`)

- `dispatchTaskEvent(event)` — 从任意 SSE 流发布事件
- `subscribeTaskEvents(fn)` — 订阅，返回 unsubscribe 函数
- `isTaskEventType(type)` — 判断是否 task 事件

**ChannelView** 中当 SSE 事件类型为 `task_started`, `step_started`, `step_completed`, `task_completed`, `approval_required` 时：
1. 调用 `dispatchTaskEvent(event)` 通知所有订阅者
2. 跳过渲染（不产生聊天气泡）
3. 保持原 Chat 完全不受影响

## TaskListView 功能

- **任务卡片列表**：标题、状态徽章、描述、创建者/时间、进度条
- **状态筛选**：全部 / 待处理 / 执行中 / 已完成 / 失败
- **创建任务**：弹窗，输入标题、描述、优先级
- **删除任务**：确认弹窗，级联删除步骤和记录
- **点击任务**：进入 `TaskDetailView`

## TaskDetailView 功能

- **头部**：返回按钮、标题、状态徽章
- **操作按钮**：根据当前状态动态展示
  - CREATED → 开始规划
  - PLANNING → 执行任务 / 标记失败
  - EXECUTING → 暂停 / 标记完成 / 标记失败
  - PAUSED → 继续执行 / 取消任务
  - FAILED → 重新规划
- **信息卡片**：描述、创建者、优先级、时间
- **进度条**：已完成/总步骤数 + 百分比
- **步骤时间线**：step number → 状态图标 → 目标 → 输出 → 错误 → 时间戳
  - 运行中步骤高亮 + 旋转动画
  - 失败步骤红色错误信息
- **审批列表**：展示审批请求状态
- **刷新按钮**：手动重新加载
- **SSE 事件更新**：订阅 task events，自动刷新

## UV (不修改 Chat，不修改后端)
