# V2.7 Phase C — Memory Intelligence Layer Report

> **日期**: 2026-07-08
> **目标**: 增加 Memory 分析能力，从历史 ExecutionResult 中归纳经验规则（Insights），辅助 Planner 决策

---

## 1. 新增内容

### 1.1 核心洞察引擎 (`services/memory/memory_insight.py`)

| 组件 | 说明 |
|------|------|
| `InsightType` 枚举 | `SUCCESS_PATTERN`, `FAILURE_PATTERN`, `OPTIMIZATION`, `RISK_WARNING`，带优先级排序 |
| `MemoryInsight` 数据类 | 纯 Python dataclass，无 ORM 依赖；`to_dict()`/`from_dict()` 序列化 |
| `TaskResultSnapshot` | 轻量适配器，将任意 dict（来自 SQL 查询）转换为引擎可读格式 |
| `MemoryInsightEngine` | 纯规则引擎，4 条规则驱动，**零 LLM 调用** |
| `make_risk_insight()` | 静态工厂方法，外部 hook 用于生成 RISK_WARNING 类型洞察 |

**4 条规则**:

| 规则 | 触发条件 | InsightType | Confidence |
|------|---------|-------------|------------|
| 稳定成功 | `outcome=SUCCESS` + `overall_quality >= 0.8` + 无 failure_category | `SUCCESS_PATTERN` | `min(quality, 0.95)` |
| 执行失败 | `outcome=FAILURE` 或 failure_category 非空 | `FAILURE_PATTERN` | 0.7 |
| 优化建议 | `estimated_cost >= 500µ$` 或 `total_tokens >= 10000` | `OPTIMIZATION` | 0.6 |
| 风险警告 | 外部调用（策略拦截/审批拒绝场景） | `RISK_WARNING` | 可配置 |

### 1.2 持久化层 (`services/memory/memory_insight_store.py`)

- 独立表 `memory_insights`，**不使用 Task Models**
- 纯 raw SQL（aiosqlite），通过既有 async engine 操作
- 自动建表 + 索引（type, source_task_id, created_at）

| 方法 | 说明 |
|------|------|
| `create_insight()` | 单条写入 |
| `create_insights_batch()` | 批量写入（单事务） |
| `list_insights(task_id)` | 按任务列出，优先级+时间排序 |
| `list_by_type(type)` | 按 InsightType 过滤 |
| `get_recent(limit)` | 最近 N 条 |
| `get_by_ids(ids)` | 按 ID 批量查询 |
| `search_insights(keyword)` | 关键词搜索（LIKE） |
| `stats()` | 各类型统计 |
| `prune(older_than_days)` | 自动清理旧记录 |

### 1.3 编排服务 (`services/memory/memory_intelligence.py`)

`MemoryIntelligenceService` 统一入口：

- `process_task_completion(db, task_id)` — 主流程：
  1. `TaskStateManager.list_results_by_task()` 加载 ExecutionResult
  2. `TaskResultSnapshot` 包装
  3. `MemoryInsightEngine.generate_insights()` 分析
  4. `MemoryInsightStore.create_insights_batch()` 持久化
  5. 全部异常被捕获，不 Propagate
- `add_risk_insight(task_id, ...)` — 直接写入 RISK_WARNING（用于 hook）
- `list_insights()` / `search_insights()` — 查询代理
- `enabled` 开关，全局可禁用

## 4. 数据流

```
Task completed/failed event
    │
    ▼
MemoryEventHandler.on_task_completed/failed()
    │  asyncio.ensure_future()
    ▼
MemoryIntelligenceService.process_task_completion(db, task_id)  ◄── self-contained DB session
    │
    ├─1. TaskStateManager.list_results_by_task(task_id)
    │      ▼
    │   List of ExecutionResult dicts
    │
    ├─2. TaskResultSnapshot adapter × N
    │
    ├─3. MemoryInsightEngine.generate_insights(snapshots)
    │      ▼
    │   List of MemoryInsight (SUCCESS/FAILURE/OPTIMIZATION)
    │
    └─4. MemoryInsightStore.create_insights_batch(insights)
           ▼
        memory_insights table
```

```
Planner invokes
    │
    ▼
PlannerContextBuilder.build()
    │
    ├─ _collect_intelligence()  ← V3.1 Memory Intelligence (semantic search)
    ├─ _collect_insights()      ← V2.7 Phase C (structured rules, from memory_insights table)
    │      ▼
    │   merge → intelligence_context section
    │
    └─ PlannerContext.intelligence_context  ← consumed by Planner
```
