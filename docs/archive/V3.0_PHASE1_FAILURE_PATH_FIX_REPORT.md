# V3.0 Phase 1 — Task Runtime Failure Path 修复报告

## 概述

修复 Phase 1 审计发现的 3 个问题（P0/P1/P2），覆盖 `task_executor.py` 和 `retry_policy.py`。

## 修复内容

### P0 — PolicyBlockedError 调用签名错误

**问题：**

```python
# task_executor.py 旧代码 (line 159-161)
await self.result_handler.handle_task_completion(
    db, task, step, events, success=False,  # TypeError!
)
```

`handle_task_completion()` 的签名是 `(db, task)` — 只接受 2 个参数。调用传了 5 个参数，运行时必然触发 `TypeError`。

**修复：**

```python
# 改为走 handle_task_failure
step = await self.state.transition_step_status(db, step, TaskStepStatus.FAILED)
execution = await self.result_handler.record_execution(...)
await self.result_handler.update_execution_result(db, execution, error=f"Policy blocked: ...", ...)
await self.result_handler.handle_task_failure(db, task)
```

- 用 `transition_step_status()` 替代直接 `step.status = FAILED`（同时修复 P1）
- 创建 ExecutionRecord 记录 policy 拒绝原因（满足"写 execution/result"要求）
- 调用 `handle_task_failure(db, task)` 正确更新 task → FAILED

### P1 — 直接赋值 step.status 绕过状态机

**问题：**

```python
# task_executor.py 旧代码 (line 156)
step.status = TaskStepStatus.FAILED
db.add(step)
```

直接赋值跳过 `transition_step_status()`，不触发 event、不记录 trace、不保留 retry 信息。

**修复：** `step = await self.state.transition_step_status(db, step, TaskStepStatus.FAILED)`

已确认 `task_executor.py` 中所有 `.status =` 赋值均已替换为 `transition_step_status()` 调用：
- line 156 → 替换（PolicyBlockedError 路径）
- line 262、405 原本就是 `transition_step_status()` 调用

### P2 — Retry 语义不一致

**问题 A — `task_executor.py`：**

```python
# 旧代码 (line 209)
max_attempts = self.retry_policy.max_retries  # max_retries=2 → 只执行2次
```

`max_attempts = max_retries` 导致 `max_retries=2` 时只有 1 次 initial + 0 次 retry，语义错误。

**修复：** `max_attempts = self.retry_policy.max_retries + 1`

**问题 B — `retry_policy.py`：**

```python
# 旧代码 (line 116)
if unit.attempt >= self.max_retries:  # attempt=2 时 max_retries=2 → 过早 ABORT
```

`>=` 导致第 2 次 attempt（第 1 次 retry）就被 ABORT。

**修复：** `if unit.attempt > self.max_retries:`

**修正后语义（`max_retries=2`）：**

| Attempt | 含义 | decide() 结果 |
|---------|------|---------------|
| 1 | Initial | RETRY (失败时) |
| 2 | 1st retry | RETRY (失败时) |
| 3 | 2nd retry | ABORT |

共 3 次 attempt（initial + 2 retries），符合 `max_retries` 定义。
